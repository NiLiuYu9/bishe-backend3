# 动态路由实现计划

## 一、需求分析

### 当前问题

1. `api_info` 表中的 `endpoint` 字段只存储路径（如 `/weather`），没有目标服务器地址
2. Gateway 使用静态配置的路由规则，只能转发到固定服务
3. 无法支持多个不同域名/端口的 API 服务

### 目标

实现网关从 `api_info` 表读取不同的 URL，动态分发到对应的目标服务器（如 `localhost:1234`、`localhost:2345` 等）

***

## 二、数据库修改

### 2.1 修改 `api_info` 表结构

新增字段 `target_url`，存储目标服务器的基础地址：

```sql
ALTER TABLE `api_info` ADD COLUMN `target_url` VARCHAR(255) NULL COMMENT '目标服务器地址(如http://localhost:1234)' AFTER `endpoint`;
```

### 2.2 示例数据

| id | name  | endpoint    | target\_url             | method |

| 1  | 天气API | /weather    | <http://localhost:1234> | GET    |
| 2  | 用户API | /user/info  | <http://localhost:2345> | GET    |

## 五、关键代码设计

### 5.1 DynamicRouteFilter 核心逻辑

```java
@Component
@Slf4j
public class DynamicRouteFilter implements GlobalFilter, Ordered {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String targetUrl = exchange.getAttribute("targetUrl");
        
        if (StrUtil.isNotBlank(targetUrl)) {
            URI uri = URI.create(targetUrl + exchange.getRequest().getURI().getPath());
            ServerHttpRequest request = exchange.getRequest().mutate().uri(uri).build();
            return chain.filter(exchange.mutate().request(request).build());
        }
        
        return chain.filter(exchange);
    }

    @Override
    public int getOrder() {
        return 2; // 在 InterfaceValidateFilter 之后执行
    }
}
```

### 5.2 InterfaceValidateFilter 修改

```java
// 在 filter 方法中添加
if (interfaceInfo != null && StrUtil.isNotBlank(interfaceInfo.getTargetUrl())) {
    exchange.getAttributes().put("targetUrl", interfaceInfo.getTargetUrl());
}
```

## 六、访问方式说明

修改后，前端访问方式：

1. **通过 Gateway 动态路由**：

   * 请求：`http://localhost:9000/mock/weather`

   * Gateway 从数据库查询 `/weather` 对应的 `targetUrl`（如 `http://localhost:1234`）

   * 转发到：`http://localhost:1234/weather`

2. **通过 Backend 服务调用**：

   * 请求：`POST http://localhost:8080/api/invoke/call`

   * Backend 从数据库获取 `targetUrl` + `endpoint`

   * 直接调用目标服务器

***

## 七、注意事项

1. **缓存优化**：可以在 Gateway 层添加 Redis 缓存，减少数据库查询
2. **健康检查**：建议添加目标服务的健康检查机制
3. **超时配置**：根据不同 API 配置不同的超时时间
4. **安全性**：`targetUrl` 应该做格式校验，防止 SSRF 攻击
